# Functional Requirements

## Navigation & State

- Multi-step navigation with forward and backward movement
- Form state persists when navigating between steps

## Completion Flow

- Final summary step allowing users to review selections before confirmation

## Responsiveness & Interaction

- Responsive layout for mobile and desktop viewports
- Hover and focus states for all interactive elements

## Validation

### Rules

- Required fields must be completed before advancing to the next step
- Email input must be in a valid format
- Steps that require a selection cannot be submitted empty
